# Contributing to BU CSE Batch-13

This is a shared academic space. Keep contributions accurate, useful, and respectful of classmates' privacy.

## Fastest way to publish useful information

Most site content lives in [`data/site-data.js`](data/site-data.js). You can add:

- an announcement or official notice;
- a verified study resource;
- a course focus, book, or study suggestion.

Read the [content editor guide](data/README.md) before editing. It shows the small data format the website uses.

## Suggested workflow

1. Fork the repository and create a focused branch.
2. Make one clear change, such as adding an official routine or a verified C programming resource.
3. Check dates, spelling, links, and source credibility.
4. Open `index.html` through a local server or GitHub Pages and test the relevant page.
5. Open a pull request with a short description of what changed and where the information came from.

## Content standards

- Prefer official university, department, or course sources for notices.
- Add the source link for deadlines, routines, fees, and admission information whenever available.
- Do not publish phone numbers, addresses, private messages, or personal information without permission.
- Keep resources legal, accessible, and relevant to the course.
- Write a concise summary so classmates understand why an item matters.

## Notes and projects

For a note or project, add a folder with a descriptive name, a short `README.md`, and attribution. Explain the course, topic, and how a student can use or run the material.
